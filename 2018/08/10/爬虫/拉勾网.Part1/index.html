表1.General公共头部
| 字段 | 说明 | 参考链接 |
—|—|—
| Request URL: https://www.lagou.com/jobs/positionAjax.json?needAddtionalResult=false | 请求的域名 | |
| Request Method: POST | 请求方式POST或者GET | |
| HTTP状态码 | 请求返回状态为200，代表服务器成功返回网页 | |
| Remote Address | 请求的远程地址 | |
| Referrer Policy | Referrer用以指定该请求是从哪个页面跳转页来的，常被用于分析用户来源等信息，Referrer Policy过滤Referrer报头内容的标准 | |
表2.response headers响应头
| 字段 | 说明 | |
—|—|—
| HTTP/1.1 200 OK | 响应状态码，200正常应答 | |
| Server: nfs/1.0.0.2 | 服务器名，Servlet一般不设置这个值，而是由Web服务器自己设置 | |
| Date: Fri, 10 Aug 2018 00:52:48 GMT | 生成消息的具体日期时间 | |
| Content-Type: application/json;charset=UTF-8 | 响应的对象类型和字符集 | |
| Transfer-Encoding: chunked | 数据传输的方式 | |
| Connection: keep-alive | 当浏览器与服务器通信时对于长连接如何进行处理：close/keep-alive | |
| REQUESTID: 33ed4ef9-afd6-4df1-96ec-7410fe5c6dcd | 请求ID | |
| Pragma: no-cache | Pragma: no-cache兼容http1.0，Cache-Control: no-cache是http1.1 | |
| Cache-Control: no-cache, no-store, max-age=0 | 告诉浏览器或者其他客户，什么环境可以安全地缓存文档 | |
| Expires: Thu, 01 Jan 1970 00:00:00 GMT | 在什么时候认为文档已过期从而不再缓存它 | |
| Content-Language: zh-CN | 响应的语言 | |
| Cache-Control: no-cache | 告诉浏览器或者其他客户，什么环境可以安全地缓存文档 | |
表3.request headers请求头
| 字段 | 说明 | |
—|—|—
| POST /jobs/positionAjax.json?needAddtionalResult=false HTTP/1.1 | 请求方式 文件名 http版本号 | |
| Host: www.lagou.com | 请求的服务器URL | |
| Connection: keep-alive | 当浏览器与服务器通信时对于长连接如何进行处理 | |
| Content-Length: 55 | 发送给HTTP服务器的长度 | |
| Origin: https://www.lagou.com | 最初请求是从哪里发起的,origin只用于Post请求 | |
| X-Anit-Forge-Code: 0 | | |
| User-Agent: Mozilla/5.0 (Windows NT 6.1; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/68.0.3440.84 Safari/537.36 | 客户端浏览器使用的操作系统和浏览器的版本和名称 | |
| Content-Type: application/x-www-form-urlencoded; charset=UTF-8 | 浏览器接收的内容类型、字符集 | |
| Accept: application/json | 浏览器接收的媒体类型 application/json | |
| X-Requested-With: XMLHttpRequest | 判断request来自Ajax请求(异步)还是传统请求(同步) | |
| X-Anit-Forge-Token: None | 动态token，防御伪查找请求 | |

| Accept-Encoding: gzip, deflate, br | 浏览器申明自己接收的编码方式 | |
| Accept-Language: zh-CN,zh;q=0.9,en;q=0.8 | 浏览器申明自己接收的语言 | |
| Cookie: xxx | 向服务器返回cookie，这些cookie是之前服务器发给浏览器的 | |
